What Was Changed

Added a comments section to blog articles.

Added a max-width for all text.

Added a div container around all image tags.

Adjusted hero images on blog specific pages as users said it that they couldn't view it properly.

Changed colour of text on contact us section on landing page as the contrast was too low.

How I Changed It

Created a comments button. When the user clicks on it the form will display. I re-used the same styling as the contact form. Connected form to the comments form on wordpress and collected their comment data using the Flamingo plugin as well as added HTML validation.

I set the max-width for the container surrounding the text.

I previously had the image size set on the image tag. I now gave the image a width and height of 100% and then specified a width and height for the surrounding container. I also gave the image the object-fit property.

All I did for this fix was to adjust the viewport height for the section and adjust the background image properties.

I just changed the text colour to my black colour so that it would stand out better against the background colour.

Why I Did It That Way

I chose to display the form only when the user clicks so that it wouldn't clutter the page and the user could focus on the story. I think there was another way to post comments to wordpress but I could not get it working so I chose to use a contact form as it would look the same as well as store the data on the back-end.

If a container contained multiple text elements then they would all have a max-width if I set it on the container.

This would ensure that the images retain their aspect ratio when changing the viewport size.

This was a simple fix but allowed the user to have a more pleasurable experience when viewing blogs on different viewport sizes.

I chose to change the colour to the black as I knew the contrast would be much higher against that background. This would improve the accessibility of the site.
